# local AI coding assistant

Voici un **résumé clair en français** de l’article :

---

## 🧠 Objectif de l’article

L’article explique comment intégrer **VS Code avec Ollama** pour utiliser une **IA locale** comme assistant de programmation, sans dépendre du cloud.

👉 Idée clé : avoir un assistant IA directement dans ton IDE, tout en gardant **confidentialité et contrôle**.

---

## ⚙️ Étapes principales

### 1. Installer Ollama

* Télécharger Ollama (Windows / macOS) ou utiliser une commande sur Linux.
* Installer un modèle (ex : `codellama`) pour générer du code. ([The New Stack][1])

---

### 2. Installer VS Code

* Installation classique selon ton OS (exécutable ou package Linux). ([The New Stack][1])

---

### 3. Ajouter l’extension “Continue”

* Installer l’extension `Continue` dans VS Code.
* Elle permet de connecter l’éditeur à un modèle IA. ([The New Stack][1])

---

### 4. Configurer Ollama dans VS Code

* Dans l’extension :

  * Ajouter un modèle
  * Choisir **Ollama comme provider**
  * Sélectionner un modèle local
* Exécuter les commandes pour :

  * chat
  * autocomplétion
  * embeddings ([The New Stack][1])

---

### 5. Utilisation

Une fois connecté :

* Chat avec l’IA directement dans VS Code
* Génération de code
* Debug
* Explications de concepts

👉 L’IA agit comme un **assistant de développement intégré**.

---

## ✅ Avantages mis en avant

* 🔒 **Confidentialité** : aucune donnée envoyée au cloud
* ⚡ **Fonctionne hors ligne**
* 🧩 **Flexible** : choix du modèle
* 🚀 **Productivité** : aide au code, debug, apprentissage ([The New Stack][2])

---

## ⚠️ Conseil important

L’article insiste :

> Utiliser l’IA comme **assistant**, pas comme remplacement de l’apprentissage.

---

## 🧾 Conclusion

Cette intégration permet de transformer VS Code en **environnement de développement augmenté par IA locale**, puissant et privé, idéal pour apprendre et coder plus efficacement.
